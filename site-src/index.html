LaTeX package for CORAS notation

CORAS diagrams.

__SUMMARY__

__DESCRIPTION__

CORAS is a model-based risk analysis method and notation. This package gives that notation a direct, reusable LaTeX authoring layer. Learn more at coras.sourceforge.net.

Get Started Download Manual PDF View on GitHub
  • Semantic diagram environments
  • Bundled CORAS icons and perspectives
  • High-level analysis tables
  • pdflatex and lualatex support

Why this package

What it gives you beyond raw TikZ drawing

Notation-first authoring

Use CORAS concepts directly instead of reconstructing the notation by hand from low-level drawing primitives.

Semantic structure

Diagram environments enforce legal CORAS vocabulary and edge relationships so invalid combinations fail early.

Reusable visual language

Bundled icons, perspectives, and packaged examples keep reports visually consistent across diagrams and tables.

What you can build

Support for the core CORAS diagram families and report tables

Quickstart

Start with Overleaf or a project-local copy

1. Copy the package assets

Upload or copy corasdiagram.sty together with the generated icons/ directory into your project.

main.tex
corasdiagram.sty
icons/
  corasdiagram-asset.pdf
  corasdiagram-stakeholder.pdf
  corasdiagram-risk.pdf
  ...

2. Load the package and use semantic environments

The package resolves its bundled icon path relative to the .sty file, so the simplest project layout just works.

__MINIMAL_SNIPPET__

Local TEXMF or CTAN/TeX Live later

For a user TEXMF installation, copy tex/latex/corasdiagram/ into your local TEXMF tree. Once the package is published through CTAN and TeX Live, you can switch to the normal package manager path while keeping the same \usepackage{corasdiagram} usage.

How it works

Compose diagrams from CORAS concepts instead of drawing them from scratch

Semantic environments and macros

Start from the diagram family you want, place concept nodes such as assets, threats, risks, treatments, and scenarios, then connect them with semantic edge macros.

__HOW_IT_WORKS_SNIPPET__

Perspectives, tables, and engine support

  • perspective=before|before-after|after swaps the packaged CORAS icon state without changing the public macro.
  • High-level analysis tables reuse the same icon language for report-style deliverables.
  • The package supports both pdflatex and lualatex.
  • The manual PDF remains the complete reference for keys, macros, and notation rules.
